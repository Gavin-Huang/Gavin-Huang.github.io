GDI+高速画图设计解析

在做离线编程交互式编辑时需要使用GDI+高速画图显示大量焊点和模型，如果模块设计不是很好就会导致交互卡顿，刷新出现闪屏等问题。下面介绍一下我解决该问题的思路。

背景介绍

GDI+

GDI+（Graphics Device Interface plus）是Windows中的一个子系统，它主要负责在显示屏幕和打印设备输出有关信息，它是一组通过C++类实现的应用程序编程接口。顾名思义，GDI+是以前版本GDI的继承者，出于兼容性考虑，Windows仍然支持以前版本的GDI，但是在开发新应用程序的时候，开发人员为了满足图形输出需要应该使用GDI+，因为GDI+对以前的Windows版本中GDI进行了优化，并添加了许多新的功能。
我们使用GDI+接口进行界面绘图，GDI+使用Pen(画笔)、Path（路径）、Image(图像)、Font（字体）、笔刷（Grapthics）等类在画布上进行绘制点、线、矩形、多边形、圆等图形，GDI+还提供三维矩阵偏移、旋转、翻转等复杂计算。如果我们只需要完成二维图像交互程序，GDI+能完美满足我们的需求。

一道算法题的解题思路

题目

  • 方程g(x,y)=（x^2+y^2-1)^2-x^2*y^2，g(x,y)=0方程 能画出一个美丽的爱心，现在给定一个点（X，Y），其中 X、Y分别属于N(μ1，σ1^2)、N(μ1，σ1^2)的正态分布， 那么这个点在这个爱心里面的概率是多少，精确到0.1（小数点后保留1位有效数，比如0.1, 0.2等）

    思路解析

  • 当时看到这个题目是一脸懵逼的，没见过这样纯数学的题目，我开始想着用数学的方法，用联合概率密度在方程所围成的图形里积分，算了一下，这个得积二元积分，先积x再积y,光算x用y来表示界限就不得了，关键图像关于y=x对称，而且xy都以平方为基元，交点有4个–||.
  • 硬算肯定是不行的，和同学讨论，得到一种完全全新的思路，可以在用程序去逼近求解，在(x,y,x+deltaX，y+deltaX)的矩形区域内，以联合概率密度 f(x,u1,σ1) * f(y,u2,σ2) 作为其概率密度，然后硬积分出在图形内的总的概率。
  • 那么现在的问题就是如何确定（x,y）是否在方程（x^2+y^2-1)^2-x^2*y^2=0所围成的图形内。首先得知道这个图形大致样子，MathLab是个好东西，画函数图像非常好用。
  • 在mathlab的命令行中使用 ezplot('(x^2+y^2-1)^2-x^2*y^2=0')

  • 使用grid on命令显示网格方便看交点位置
